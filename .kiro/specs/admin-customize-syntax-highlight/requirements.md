# Requirements Document

## Introduction

`/admin/customize` 管理画面のカスタムスクリプト・カスタム CSS・カスタム Noscript の各入力欄は、現状プレーンな `<textarea>` で実装されており、コードのシンタックスハイライトが効いていない。GROWI v5 ではこれらの欄は CodeMirror ベースの専用エディタ（`CustomScriptEditor`）で提供され、シンタックスハイライト・行番号・括弧のマッチ表示などを備えていたが、v6 系への移行時に textarea へ置き換えられ、これらの編集支援が失われた（コミット `af9df831dc`）。

本仕様は、管理者がカスタムコードを編集する際の体験を v5 相当まで復元することを目的とする。3 つの入力欄を、言語に応じたシンタックスハイライトと行番号・括弧支援を備えたコードエディタに置き換え、かつ現行の管理画面のライト/ダークテーマに追従させる。既存の保存・読み込み・その他の周辺 UI（サンプルコードの折りたたみ表示、更新ボタン等）の挙動は維持する。

## Boundary Context

- **In scope**:
  - カスタムスクリプト（JavaScript）・カスタム CSS（CSS）・カスタム Noscript（HTML）の 3 入力欄のコードエディタ化
  - 言語別シンタックスハイライト、行番号表示、括弧のマッチ表示・自動閉じ
  - 管理画面のライト/ダークテーマへの追従
  - 既存の値の読み込み・編集・保存フローの維持（非退行）
- **Out of scope**:
  - コード補完（Ctrl-Space 等の autocomplete）
  - エディタのドラッグリサイズ（v5 の jquery-ui ベース機能）
  - 入力されたカスタムコードの構文検証（lint）・エラー表示
  - `/admin/customize` の他セクション（テーマ、レイアウト、タイトル等）の変更
  - サンプルコード表示（"Example for Google Tag Manager"）の表示技術の刷新（現行の表示は維持する）
- **Adjacent expectations**:
  - テーマ状態は既存の管理画面テーマ機構（`useNextThemes` 等の `resolvedTheme`）から取得する前提であり、本仕様は新たなテーマ切替 UI を提供しない。
  - 保存 API・保存データ形式（`customizeScript` / `customizeCss` / `customizeNoscript` 文字列）は既存のまま変更しない。

## Requirements

### Requirement 1: カスタムコード入力欄のシンタックスハイライト表示

**Objective:** 管理者として、カスタムスクリプト・CSS・Noscript を編集する際に言語に応じた色分け表示がほしい。そうすることでコードの構造を把握しやすく、記述ミスに気付きやすくなる。

#### Acceptance Criteria

1. When 管理者が `/admin/customize` を開いたとき, the カスタマイズ設定画面 shall カスタムスクリプト入力欄の内容を JavaScript のシンタックスハイライトで表示する。
2. When 管理者が `/admin/customize` を開いたとき, the カスタマイズ設定画面 shall カスタム CSS 入力欄の内容を CSS のシンタックスハイライトで表示する。
3. When 管理者が `/admin/customize` を開いたとき, the カスタマイズ設定画面 shall カスタム Noscript 入力欄の内容を HTML のシンタックスハイライトで表示する。
4. When 管理者が入力欄でコードを入力・編集したとき, the カスタマイズ設定画面 shall 入力中の内容に対してもシンタックスハイライトを継続的に反映する。
5. If 入力欄の内容が空であるとき, the カスタマイズ設定画面 shall エラーを発生させず空の状態のエディタを表示する。

### Requirement 2: コード編集支援（行番号・括弧）

**Objective:** 管理者として、コードエディタとしての基本的な編集支援がほしい。そうすることで v5 相当の編集効率でカスタムコードを記述できる。

#### Acceptance Criteria

1. While いずれかのカスタムコード入力欄が表示されているとき, the カスタマイズ設定画面 shall 各行に行番号を表示する。
2. When 管理者がカーソルを括弧（`()`, `{}`, `[]`）付近に置いたとき, the カスタマイズ設定画面 shall 対応する括弧をマッチ表示する。
3. When 管理者が開き括弧を入力したとき, the カスタマイズ設定画面 shall 対応する閉じ括弧を自動補完する。

### Requirement 3: テーマ追従

**Objective:** 管理者として、管理画面のライト/ダークテーマに合わせてコードエディタの配色が切り替わってほしい。そうすることで画面全体の見た目が統一され、視認性が保たれる。

#### Acceptance Criteria

1. While 管理画面がライトテーマで表示されているとき, the カスタマイズ設定画面 shall コードエディタをライトテーマ相当の配色で表示する。
2. While 管理画面がダークテーマで表示されているとき, the カスタマイズ設定画面 shall コードエディタをダークテーマ相当の配色で表示する。
3. When 管理者が管理画面のテーマを切り替えたとき, the カスタマイズ設定画面 shall コードエディタの配色を切替後のテーマに追従させる。

### Requirement 4: 既存の編集・保存挙動の維持（非退行）

**Objective:** 管理者として、エディタ化されても従来どおり値の読み込み・編集・保存ができてほしい。そうすることで既存の運用が壊れない。

#### Acceptance Criteria

1. When 管理者が `/admin/customize` を開いたとき, the カスタマイズ設定画面 shall 保存済みのカスタムスクリプト・CSS・Noscript の値を各エディタの初期内容として読み込む。
2. When 管理者がエディタの内容を変更して更新ボタンを押したとき, the カスタマイズ設定画面 shall 変更後の内容を既存の保存 API 経由で保存する。
3. When 保存が成功したとき, the カスタマイズ設定画面 shall 従来どおり成功トーストを表示する。
4. If 保存処理でエラーが発生したとき, the カスタマイズ設定画面 shall 従来どおりエラートーストを表示する。
5. While 設定値の取得に失敗している状態（`retrieveError` あり）であるとき, the カスタマイズ設定画面 shall 更新ボタンを無効化した従来どおりの挙動を維持する。
6. The カスタマイズ設定画面 shall 保存されるデータ形式（各項目の文字列）を従来から変更しない。

### Requirement 5: 周辺 UI の維持と一貫性

**Objective:** 管理者として、エディタ化後も 3 つの入力欄が一貫した見た目・操作感で、既存の補足表示が保たれてほしい。そうすることで学習コストなく利用できる。

#### Acceptance Criteria

1. The カスタマイズ設定画面 shall カスタムスクリプト・CSS・Noscript の 3 入力欄で一貫したコードエディタの見た目と操作感を提供する。
2. Where カスタムスクリプト欄およびカスタム Noscript 欄にサンプルコードの折りたたみ表示（"Example for Google Tag Manager"）が存在する場合, the カスタマイズ設定画面 shall 当該サンプル表示を従来どおり維持する。
3. The カスタマイズ設定画面 shall 各セクションの見出し・説明文・更新ボタンなど入力欄以外の既存 UI 構成を維持する。
