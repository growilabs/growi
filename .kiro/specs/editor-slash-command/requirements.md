# Requirements Document

## Introduction

GROWI の Markdown エディタにスラッシュコマンド機能を追加する。ユーザーがエディタのテキスト入力中に `/` を入力するとコマンドメニュー（ポップアップ）が表示され、続けて文字を入力すると候補が絞り込まれ、キーボードまたはクリックで選択すると対応する Markdown 要素（見出し・リスト・コードブロック・テーブル等）が即座に挿入される。これにより、ツールバーのボタンを探す操作なしに、執筆フローを止めず高速に要素を挿入できるようにする。

背景・スコープ・依存関係の詳細は [brief.md](./brief.md) を参照。

## Boundary Context

- **In scope**:
  - エディタのテキスト入力中に `/` でコマンドメニューを表示する
  - 入力文字によるコマンドの絞り込み、キーボード操作（矢印 / Enter / Escape）とクリックによる選択
  - 基本 Markdown 要素の挿入: 見出し（H1–H3）、箇条書きリスト、番号付きリスト、チェックボックス（タスクリスト）、引用、コードブロック、テーブル（プレーンな Markdown テーブルを即挿入）
  - 選択せずにメニューを閉じた場合に入力テキストを保持する誤爆防止挙動
  - コマンドのラベル・説明の多言語表示
- **Out of scope**:
  - GROWI 固有の拡張要素（drawio、math、lsx、テンプレート等）の挿入コマンド（将来拡張）
  - テーブル挿入時の表ビルダー（行列数を指定するモーダル）起動（将来拡張）
  - スラッシュコマンドの並び順・表示内容のユーザーカスタマイズ
  - エディタ外（閲覧モード等）での発火
  - 単語の途中（空白以外の文字の直後）での発火（誤爆防止のため非対応）
  - インライン要素の挿入（将来拡張。MVP のコマンドはすべてブロックレベル要素）
- **Adjacent expectations**:
  - アプリ全体のホットキー `/`（ページ検索）は従来どおり動作する。スラッシュコマンドはエディタのテキスト入力中のみ発火し、グローバル `/` の挙動を変えない。
  - 既存の絵文字オートコンプリート（`:` トリガー）と同時に存在しても互いに干渉しない。
  - 協調編集（複数ユーザー同時編集）環境でも、挿入が他ユーザーの編集との整合性を壊さない。
  - キーバインド体系（Ctrl+B 等）には影響を与えない。

## Requirements

### Requirement 1: スラッシュコマンドメニューの起動

**Objective:** As an エディタで執筆するユーザー, I want `/` を入力するだけでコマンドメニューを開きたい, so that ツールバーを探さずに要素挿入を開始できる

#### Acceptance Criteria

1. When ユーザーがエディタのテキスト領域で、行頭（先頭の空白文字を除いた行の先頭）または空白文字の直後に `/` を入力する, the スラッシュコマンド機能 shall コマンドメニューをカーソル位置の近傍に表示する
2. If `/` が空白以外の文字の直後（単語の途中、例: `foo/`）に入力される, then the スラッシュコマンド機能 shall コマンドメニューを表示しない
3. The スラッシュコマンド機能 shall メニュー内の各コマンドについて、識別できるラベルと内容を説明する短いテキストを表示する
4. While コマンドメニューが表示されている, the スラッシュコマンド機能 shall いずれか1つのコマンドを初期ハイライト状態にする

### Requirement 2: コマンドの絞り込み

**Objective:** As an ユーザー, I want 入力した文字でコマンドを絞り込みたい, so that 目的のコマンドへ素早く到達できる

#### Acceptance Criteria

1. When ユーザーがメニュー表示中に `/` に続けて文字を入力する, the スラッシュコマンド機能 shall 入力文字列に一致するコマンドのみを候補として表示する
2. The スラッシュコマンド機能 shall コマンドの一致判定を大文字・小文字を区別せずに行う
3. When ユーザーが文字を削除して入力文字列が変化する, the スラッシュコマンド機能 shall 候補一覧を最新の入力に合わせて更新する
4. If 入力文字列に一致するコマンドが存在しない, then the スラッシュコマンド機能 shall メニューを閉じ、入力済みのテキストをそのまま残す

### Requirement 3: コマンドの選択と要素の挿入

**Objective:** As an ユーザー, I want 候補を選択して対応する要素を挿入したい, so that 手早く Markdown 要素を書き始められる

#### Acceptance Criteria

1. When ユーザーが上下矢印キーを押す, the スラッシュコマンド機能 shall ハイライトされる候補を移動する
2. When ユーザーが Enter キーで候補を確定する、またはマウスで候補をクリックする, the スラッシュコマンド機能 shall 対応する Markdown 要素を挿入し、入力した `/` とそれに続くクエリ文字列を削除する
3. When ユーザーがテーブルコマンドを選択する, the スラッシュコマンド機能 shall ヘッダ行と区切り行を含む空の Markdown テーブルを挿入する
4. When 要素が挿入される, the スラッシュコマンド機能 shall ユーザーが続けて入力できる位置にカーソルを配置する
5. When 要素が挿入された直後にユーザーが取り消し（undo）操作を行う, the スラッシュコマンド機能 shall 挿入された要素を元の状態に戻す
6. When 要素を挿入する際に `/` が行の途中（同じ行に先行する非空白テキストがある位置）にある, the スラッシュコマンド機能 shall ブロック要素を新しい行に挿入し、先行するテキストを壊さない

### Requirement 4: メニューのキャンセルと誤爆防止

**Objective:** As an ユーザー, I want コマンドを使わないときは通常どおり `/` を入力したい, so that 意図しない要素挿入で執筆が妨げられない

#### Acceptance Criteria

1. When ユーザーが Escape キーを押す, the スラッシュコマンド機能 shall メニューを閉じ、入力した `/` テキストをそのまま残す
2. When ユーザーがメニューの外をクリックする、またはエディタからフォーカスが外れる, the スラッシュコマンド機能 shall メニューを閉じる
3. When ユーザーがコマンドを選択せずに、入力した `/` テキストに続けて空白を入力する, the スラッシュコマンド機能 shall 要素を挿入せず、入力されたテキストを変更しない

### Requirement 5: 提供するコマンドセット（初期リリース）

**Objective:** As an ユーザー, I want よく使う基本要素をスラッシュコマンドから挿入したい, so that 主要な Markdown 要素を素早く入力できる

#### Acceptance Criteria

1. The スラッシュコマンド機能 shall 次の要素を挿入するコマンドを提供する: 見出し H1、見出し H2、見出し H3、箇条書きリスト、番号付きリスト、チェックボックス（タスクリスト）、引用、コードブロック、テーブル
2. When ユーザーが見出し・リスト・引用・チェックボックスのコマンドを選択する, the スラッシュコマンド機能 shall 現在の行に対応する Markdown 記法のプレフィックスを付与する
3. When ユーザーがコードブロックのコマンドを選択する, the スラッシュコマンド機能 shall コードを入力できる空のコードブロックを挿入する
4. Where 拡張要素（drawio、math、lsx、テンプレート等）, the スラッシュコマンド機能 shall 本リリースではコマンドを提供しない

### Requirement 6: 既存のエディタ機能との共存

**Objective:** As an ユーザーおよび運用者, I want 既存のショートカットやエディタ機能が従来どおり動作してほしい, so that スラッシュコマンド追加による回帰が起きない

#### Acceptance Criteria

1. While エディタのテキスト領域にフォーカスがない, when ユーザーが `/` を押す, the アプリケーション shall 従来どおりページ検索を起動し、スラッシュコマンドメニューを表示しない
2. The スラッシュコマンド機能 shall 既存の絵文字オートコンプリート（`:` トリガー）と同時に有効であっても、互いの表示・動作を妨げない
3. While 複数のユーザーが同一ページを協調編集している, when スラッシュコマンドにより要素が挿入される, the スラッシュコマンド機能 shall 他ユーザーの編集内容との整合性を保つ
4. The スラッシュコマンド機能 shall 既存のキーバインド（太字 Ctrl+B 等）の動作を変更しない

### Requirement 7: コマンド表示の多言語対応

**Objective:** As an 異なる表示言語を使うユーザー, I want コマンドのラベルと説明が自分の言語で表示されてほしい, so that 内容を理解してコマンドを選べる

#### Acceptance Criteria

1. The スラッシュコマンド機能 shall コマンドのラベルと説明を、ユーザーの表示言語設定に従って表示する
2. If あるコマンドのラベルまたは説明が現在の表示言語向けに用意されていない, then the スラッシュコマンド機能 shall 既定言語のテキストを表示する
3. The スラッシュコマンド機能 shall GROWI がサポートする全ロケール（`en_US` / `ja_JP` / `fr_FR` / `ko_KR` / `zh_CN`）に対して各コマンドの `slash_command.*` ラベル・説明キーを提供する
